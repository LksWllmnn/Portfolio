Bachelorthesis - Remote Rendering

Since the beginning of September 2022 I am working on a remote rendering component in Unity for the application MARBLE which runs on the HoloLens2. The goal of this work is to show if it is possible to create a component for the application that implements the principle of remote rendering and what difficulties it can cause. The concept of remote rendering describes that a complex model is rendered on an outsourced computing unit and the result is transferred to the client so that the client does not have to bear the computing load for the rendering process.

I work with the MRTK and the Mixed Reality WebRTC Package from Microsoft. In the practical part of the work, I focus on which data has to be streamed, how to build up the scene so that a 3D image is created from the streamed data, how to get the data from the depth buffer, transfer it and integrate it again, and chromakeying so that the object integrates well into the scene.

Tools: Unity, C#, HLSL

Shader Solarsystem

Like Realtime BeerPhong, the Shader Solar System was created in the Realtime Computer Graphics module. The task was to get acquainted with the Shadertoy tool and develop a shader within a week. Within two long days I designed the idea and implemented the shader. I concentrated on the shadows of the planets to make them look like 3D models although they exist only on pixel basis.

What I find fascinating about shader coding is that the graphics card can really calculate the information for each pixel in an incredibly short time.
